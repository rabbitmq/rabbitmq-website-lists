Presumably the management database process is getting overwhelmed and 
struggling to keep up. We already prioritise this process, and we can't 
really raise the priority further.

Things you could do:

* Switch off fine-grained statistics. If you don't need message rates 
that will cut the amount of work the management database is doing.

* Run the management plugin on a separate cluster node that isn't used 
for messaging.

Cheers, Simon
