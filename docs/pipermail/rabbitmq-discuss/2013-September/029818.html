>            Requires: libodbc.so.2()(64bit)
>            Requires: tk

Make sure your package manager is set up in such a way that these dependencies can be installed. A correctly installed base OS should be able to do this.

Alternatively select one of the other providers of an Erlang package which does not depend on these packages, e.g. the package from VMware which is monolithic and more up-to-date:
http://www.rabbitmq.com/install-rpm.html

I would use "--skip-broken" only if no other option could possibly work.
While the broker will run because it does not depend on tk or libodbc, you should avoid broken packages.
