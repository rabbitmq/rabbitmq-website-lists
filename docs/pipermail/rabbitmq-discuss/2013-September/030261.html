Recent versions of RabbitMQ required all clustered nodes to be upgraded
simultaneously. Make sure this is not the problem you are running into
by ensuring that the beam files are identical across all clustered nodes.

> The version of elang is the same on both nodes however (This was
> verified with the command "erl").

Check whether any servers have multiple instances of Erlang installed.
Your check would not be valid if multiple instances are present.
