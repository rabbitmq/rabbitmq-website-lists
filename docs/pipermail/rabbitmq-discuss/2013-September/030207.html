On 17/09/13 09:08, Haster wrote:

> Do you have any idea what is the problem? And in that cases server can send
> RST packets? 

If the broker was responsible for forcing the connection to close then
the reason will be in the logfile. Check in the regular and sasl logfiles.

If the TCP connection is mediated by an intelligent network device then
that intermediary could be responsible for the RST. Try to rule that out
if it is a possibility.
