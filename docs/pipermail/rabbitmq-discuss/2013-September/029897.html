Sync of queues after a failure wasn't released until 3.1.  Mirroring has been around a while but on a failure a node when rejoining the cluster comes on as a slave, assumes its message state is invalid and drops all persistent messages.  Please read the docs - this is very clearly stated in ha and clustering documentation.
