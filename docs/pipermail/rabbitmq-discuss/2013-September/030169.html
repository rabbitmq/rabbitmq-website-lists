ramkkk:

> how i can found the performance is slow in the rabbit mq, is it based on the
> message_rates ??

Are you trying to monitor a Rabbit node/cluster and want to alert on
message rates being lower than some expected value?

HTTP API can help with that:
http://hg.rabbitmq.com/rabbitmq-management/raw-file/3646dee55e02/priv/www-api/help.html

Note that those rates indicate throughput, which is only one metric.
Some systems care much more about latency.
