<div>
                    I have tagged and pushed 0.9.13 to pypi. 0.9.13 is primarily a bug fix release but contains several major changes, the most substantial of which is dropping support for Python 2.4 and Python 2.5. These versions have been broken in Pika for the past few releases and as of late I'm receiving more inquiries for Python 3 support than I have had for older versions.
                </div><div><br></div><div>The documentation has received a bit of an overhaul as well: http://pika.readthedocs.org</div><div><br></div><div><div>**Major Changes**</div><div><br></div><div>- IPv6 Support with thanks to Alessandro Tagliapietra for initial prototype</div><div>- Officially remove support for &lt;= Python 2.5 even though it was broken already</div><div>- Drop pika.simplebuffer.SimpleBuffer in favor of the Python stdlib collections.deque object</div><div>- New default object for receiving content is a "bytes" object which is a str wrapper in Python 2, but paves way for Python 3 support</div><div>- New "Raw" mode for frame decoding content frames (#334) addresses issues #331, #229 added by Garth Williamson</div><div>- Connection and Disconnection logic refactored, allowing for cleaner separation of protocol logic and socket handling logic as well as connection state management</div><div>- New "on_open_error_callback" argument in creating connection objects and new Connection.add_on_open_error_callback method</div><div>- New Connection.connect method to cleanly allow for reconnection code</div><div>- Support for all AMQP field types, using protocol specified signed/unsigned unpacking</div><div><br></div><div>**Backwards Incompatible Changes**</div><div><br></div><div>- Method signature for creating connection objects has new argument "on_open_error_callback" which is positionally before "on_close_callback"</div><div>- Internal callback variable names in connection.Connection have been renamed and constants used. If you relied on any of these callbacks outside of their internal use, make sure to check out the new constants.</div><div>- Connection._connect method, which was an internal only method is now deprecated and will raise a DeprecationWarning. If you relied on this method, your code needs to change.</div><div>- pika.simplebuffer has been removed</div><div><br></div><div>**Bugfixes**</div><div><br></div><div>- BlockingConnection consumer generator does not free buffer when exited (#328)</div><div>- Unicode body payloads in the blocking adapter raises exception (#333)</div><div>- Support "b" short-short-int AMQP data type (#318)</div><div>- Docstring type fix in adapters/select_connection (#316) fix by Rikard Hultén</div><div>- IPv6 not supported (#309)</div><div>- Stop the HeartbeatChecker when connection is closed (#307)</div><div>- Unittest fix for SelectConnection (#336) fix by Erik Andersson</div><div>- Handle condition where no connection or socket exists but SelectConnection needs a timeout for retrying a connection (#322)</div><div>- TwistedAdapter lagging behind BaseConnection changes (#321) fix by Jan Urbański</div><div><br></div><div>**Other**</div><div><br></div><div>- Refactored documentation</div><div>- Added Twisted Adapter example (#314) by nolinksoft</div></div><div><br></div><div>As always, please report bugs or other issues at&nbsp;https://github.com/pika/pika/issues</div><div><br></div><div>Cheers,</div><div><br></div><div>Gavin</div>
