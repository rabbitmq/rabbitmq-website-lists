<p class="MsoNormal">We are using RabbitMQ 3.0.1 in a cluster behind an F5 load balancer.<o:p></o:p></p>
<p class="MsoNormal">The application works with RMQ through the VIP exposed by the F5.<o:p></o:p></p>
<p class="MsoNormal">The application is responsible for making sure that queues/exchanges are set up for the application. In other words, we don&#8217;t use a separate mechanism to set up queues and exchanges.<o:p></o:p></p>
<p class="MsoNormal">As a result of this queues may end up on a random node once created.<o:p></o:p></p>
<p class="MsoNormal">Once queues are created they remain; they are not &#8220;torn down&#8221; and re-established periodically or anything like that.<o:p></o:p></p>
<p class="MsoNormal">Most queues are HA/mirrored queues.<o:p></o:p></p>
<p class="MsoNormal"><o:p>&nbsp;</o:p></p>
<p class="MsoNormal">My question is around balancing node resources. If a queue becomes somewhat busy, can I move make a different node master for that queue (assuming a mirrored queue)? I don&#8217;t want to take down the node as I have other queues on that node;
 I just want to move the master for a particular queue, if possible.<o:p></o:p></p>
<p class="MsoNormal"><o:p>&nbsp;</o:p></p>
<p class="MsoNormal">Looking at past queries about this it appears that this might not be possible, so one answer may be that we should change how we create queues&#8230;<o:p></o:p></p>
