&gt; I would like to know, how the messages are stored in RabbitMQ server. Whether it is file level or DB level persistence?<br>
<br>
</div>RabbitMQ uses its own storage layer for messages, not an external database. Certain parts of it are pluggable<br>
and can use K/V stores such as Tokyo Cabinet.<br>
<div class="im"><br>
&gt;<br>
&gt; And please let clarify me what is DISC, RAM and MNESIA<br>
<br>
</div>Mnesia is a distributed database that RabbitMQ uses to store information about users, exchanges, queues, bindings, etc. Messages or message positions in queues are not stored in Mnesia.<br>
<br>
Mnesia can store data either in RAM or on disk, or both. You can learn about disk and RAM nodes<br>
on [1]. If you want to learn more about Mnesia, see [2], but<br>
you don&#39;t need to learn anything about it in order to use RabbitMQ.<br>
<br>
