<p class="MsoNormal">Hi,<o:p></o:p></p>
<p class="MsoNormal"><o:p>&nbsp;</o:p></p>
<p class="MsoNormal">we had a complete crash of our 3-Node 2.7.1 cluster some days ago. What I saw on the web, this issue might be related to
<a href="http://lists.rabbitmq.com/pipermail/rabbitmq-discuss/2012-April/019765.html">
http://lists.rabbitmq.com/pipermail/rabbitmq-discuss/2012-April/019765.html</a> . As such it may have been fixed in 2.8.x which we are going to migrate to. Nevertheless, I would like to ask for clarification on that if possible.<o:p></o:p></p>
<p class="MsoNormal"><o:p>&nbsp;</o:p></p>
<p class="MsoNormal">Detailed description:<o:p></o:p></p>
<p class="MsoNormal"><o:p>&nbsp;</o:p></p>
<p class="MsoNormal"><span lang="FR">OS: Linux<o:p></o:p></span></p>
<p class="MsoNormal"><span lang="FR">Erlang Version: R14A<o:p></o:p></span></p>
<p class="MsoNormal"><span lang="NL">Rabbit Version: 2.7.1<o:p></o:p></span></p>
<p class="MsoNormal">Cluster Setup: 3 Nodes, all disc mode<o:p></o:p></p>
<p class="MsoNormal">Queues: HA mode “all” and durable<o:p></o:p></p>
<p class="MsoNormal"><o:p>&nbsp;</o:p></p>
<p class="MsoNormal">The cluster used to be stable for some months with relatively low usage.
<o:p></o:p></p>
<p class="MsoNormal"><o:p>&nbsp;</o:p></p>
<p class="MsoNormal">Some days before the crash we were doing some OS related maintenance on the server. So we shut down, updated and reset each node one by one before rejoining it to the cluster. After that, some queues were not in synch any more with all
 other nodes (some synched with one, some synched with both other nodes). We regarded this as none-critical state and ignored it with respect to our already planned migration to 2.8.2.<o:p></o:p></p>
<p class="MsoNormal"><o:p>&nbsp;</o:p></p>
