<blockquote class="gmail_quote" style="margin:0 0 0 .8ex;border-left:1px #ccc solid;padding-left:1ex">Hi everyone! We&#39;ve just put in place a nightly build system for the RabbitMQ server and clients. You can download nightlies from <a href="http://www.rabbitmq.com/nightly-builds.html" target="_blank">http://www.rabbitmq.com/<u></u>nightly-builds.html</a><br>

<br>
We&#39;ve also put documentation for what will be the next version of RabbitMQ at <a href="http://next.rabbitmq.com/documentation.html" target="_blank">http://next.rabbitmq.com/<u></u>documentation.html</a>.<br>
<br>
We&#39;re evolving towards a model where we have feature releases every few months, with bugfix releases in between. It&#39;s possible there will be a 2.8.8 bugfix release, but that&#39;s not what the current nightlies are - they represent what will be RabbitMQ 3.0.<br>

<br>
So with some caveats in place:<br>
<br>
 * Obviously we do not suggest running nightlies on production systems.<br>
<br>
 * We don&#39;t always guarantee upgrades from one nightly to another will<br>
   work - you may have to reset everything and start again sometimes.<br>
<br>
 * You would also have to reset everything and start again to revert to<br>
   a stable release from a nightly.<br>
<br>
...I&#39;d invite you to have a look at what&#39;s coming, and tell use what you think.<br>
<br>
I&#39;m going to write more on the subject as we get closer to the release, but some of the major new features include:<br>
<br>
 * HA queues are configured dynamically by a new policy mechanism,<br>
   rather than by declaring them with &quot;x-ha-policy&quot;. HA-ness<br>
   can be changed at any time. HA queues are also much faster.<br>
<br>
 * Federated exchanges are now normal exchanges, with federation<br>
   similarly configured dynamically by policy. The &quot;x-federation&quot;<br>
   exchange type is no more.<br>
<br>
 * Clustering commands in rabbitmqctl have been rewritten to make<br>
   clustering easier and more robust.<br>
<br>
 * Web-STOMP and MQTT plugins are included.<br>
<br>
 * Lots of other smaller new features and bug fixes.<br>
<br>
The network protocol hasn&#39;t changed; you can run 2.x clients against 3.x server and vice versa.<br>
<br>
So please try it out, and talk to us!<br>
