Hi Kamil. There is a bug in 3.3.3 causing the the rabbitmq-server script to terminate early if you have inet_dist_listen_min / inet_dist_listen_max defined in your rabbitmq.config. This will be fixed in 3.3.4 but for the time being you can remove the "-e" from the first line of /usr/lib/rabbitmq/lib/<u></u>rabbitmq_server-3.3.3/sbin/<u></u>rabbitmq-server or switch to using RABBITMQ_DIST_PORT instead.<br>
