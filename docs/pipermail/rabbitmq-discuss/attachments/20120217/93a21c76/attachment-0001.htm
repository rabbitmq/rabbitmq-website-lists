&gt;Release notes<br>
&gt;-------------<br>
&gt;1.0.0 Release:<br>
&gt;Changed/finalized API signature:<br>
&gt; &nbsp; &nbsp;* Exposing AMQPChannel<br>
&gt; &nbsp; &nbsp;* Exposing AMQPEnvelope<br>
&gt; &nbsp; &nbsp;* Exposing more queue and exchange arguments and flags<br>
&gt; &nbsp; &nbsp;* Exposing basic.qos<br>
&gt;Added persistent connections<br>
&gt;Cleaned up codebase<br>
&gt;Fixed memory leaks and segmentation faults<br>
&gt;<br>
&gt;0.3.1 Release:<br>
&gt;Fixed bug:<br>
&gt; &nbsp; &nbsp;* 24323: Cannot get the name for auto-named reply-to queues<br>
&gt;<br>
&gt;0.3.0 Release:<br>
&gt;Fixed memory leaks in many functions (courtesy Jonathan Tansavatdi and<br>
&gt;Andy Wick)<br>
&gt;Fixed consume method to return proper values<br>
&gt;Cleaned up variable usage<br>
&gt;Fixed bugs:<br>
&gt; &nbsp; &nbsp;* 22638: Unexpected exit code 1 with AMQPQueue::consume()<br>
&gt; &nbsp; &nbsp;* 22698: AMQPQueue::consume<br>
&gt;<br>
&gt;0.2.2 Release:<br>
&gt;Made extension compatible with PHP lt 5.3 (courtesy John Skopis)<br>
&gt;Fixed wrong typing of message properties (courtesy John Skopis)<br>
&gt;<br>
&gt;0.2.1 Release:<br>
&gt;Fixed refcount decrementing bug causing segfaults.<br>
&gt;<br>
&gt;0.2.0 Release:<br>
&gt;Works with AMQP 0-8 and 0-9-1 (used by RabbitMQ 2.*)<br>
&gt;Modified AMQPConnection object:<br>
&gt; &nbsp; &nbsp;* Requires call to 'connect' method to connect (no longer connects on<br>
&gt;instantiation)<br>
&gt; &nbsp; &nbsp;* Added support for disconnect and reconnect<br>
&gt; &nbsp; &nbsp;* Added helper setters for port, host, vhost, login and password<br>
&gt;Improved consume method to block for MIN messages, and try to get MAX<br>
&gt;messages if available<br>
&gt;Fixed zval descoping bugs<br>
&gt;Fixed bugs:<br>
&gt; &nbsp; &nbsp;* 17809: Couldn't compile pecl extension under PHP 5.3<br>
&gt; &nbsp; &nbsp;* 17831: Segmentation fault when the exchange doesn't exists<br>
&gt; &nbsp; &nbsp;* 19707: AMQPQueue::get() doesn't return the message<br>
&gt; &nbsp; &nbsp;* 19840: Connection Exception<br>
&gt;<br>
&gt;Package Info<br>
&gt;-------------<br>
&gt;This extension can communicate with any AMQP spec 0-9-1 compatible<br>
&gt;server, such as RabbitMQ, OpenAMQP and Qpid, giving you the ability to<br>
&gt;create and delete exchanges and queues, as well as publish to any<br>
&gt;exchange and consume from any queue.<br>
