<p class=MsoNormal>I've been hearing from many of my clients of the desire to
use NServiceBus (an open source .NET service bus) on Linux by using Mono.<o:p></o:p></p>

<p class=MsoNormal>The thing missing is that MSMQ (the default transport used
by NServiceBus) isn't available on Linux (duh :-) )<o:p></o:p></p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal>In terms of high availability and transactional support, I
think that RabbitMQ is the best option for enabling the above.<o:p></o:p></p>

<p class=MsoNormal>The thing is if there are certain APIs that should (or
should not) be used for working with RabbitMQ in &quot;federated mode&quot; -
no central broker. Also, is there anything special that needs to be done to
flow transactions from the application level into RabbitMQ and enlisting it
into distributed transactions?<o:p></o:p></p>
