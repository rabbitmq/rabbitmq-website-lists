<br>
If I restart the RabbitMQ 3.1.1 instance while my clients are connected.<br>
Once the server is back online my clients will resume sending and the test<br>
will complete the test with success.  My clients implement a reconnect when<br>
connections are lost.<br>
<br>
When the server enters the locked state the clients do not report any<br>
connection failures or other errors. The server restart eliminates the<br>
memory high watermark.  All data makes it from the publisher to the<br>
subscriber.  Messages that might were on the server are persisted and resume<br>
sending.<br>
<br>
This is also a repeatable test.<br>
