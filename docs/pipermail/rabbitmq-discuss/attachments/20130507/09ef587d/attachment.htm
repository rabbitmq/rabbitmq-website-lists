<div>
<p class="MsoNormal">Technically it is possible to run disparate versions (the AMQP 0.9.1 protocol hasn't changed between them after all), but I would strongly advise you to use the latest version with the latest server. Not only have important bug fixes been
 introduced in the client libraries between releases, but it's possible in some cases that non-standard features&nbsp;(i.e., enhancements to or divergence from the AMQP specification)&nbsp;have been added to (or deprecated from) the broker and the client might need to
 be aware of these. Lack of support for the IMMEDIATE flag in later versions of RabbitMQ is one such example.<o:p></o:p></p>
