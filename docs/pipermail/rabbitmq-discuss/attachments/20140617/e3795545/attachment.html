<div dir="ltr"><span style="font-family:arial,sans-serif;font-size:13px">We're using the PPA for rabbit but the current debian files are not suitable for production deployments because they stop and restart the process. I understand the desire to start the process if it was not already running, but embedding the restart logic in the deb means that users have no control over the process. As it is very common to use the PPA within the context of a configuration management system (Salt, Chef, etc), it would be best to leave that decision up to the user.</span><br clear="all">
