    The rabbitmq user needs to have permissions to access the *entire*
    path. So, usually <tt>/root</tt> only allows root to access items
    within that directory (the rather bizarrely chosen "executable"
    permission on the directory). Try running "<tt>sudo -u rabbitmq
      namei /root/testca/cacert.pem</tt>" to see this in action. It's
    probably more sensible however to put the certificate somewhere
    under <tt>/etc</tt>, eg: <tt>/etc/rabbitmq</tt> or similar.<br>
