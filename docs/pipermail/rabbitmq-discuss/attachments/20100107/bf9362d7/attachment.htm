<p class=MsoNormal style='margin-bottom:12.0pt'>Alexis -<br>
<br>
Funny you should ask now.&nbsp; I've just resumed working on an AMQP interface<br>
after getting input from our team members and have been doing a lot of design<br>
work.&nbsp; Our approach is to write our own code generator that writes code to<br>
do the serialization and de-serialization.&nbsp; The interface is specified as
a Java<br>
interface, and then we will be using Java introspection to get the methods and<br>
parameters.&nbsp; The code generator will be able to make code for any
language,<br>
but all I can see happening for the short term is Java and C++.<br>
<br>
The transport is in XML.&nbsp; We use XML a lot and are comfortable with the
technology.<br>
Messages are self-describing and verifiable.&nbsp; We are leaning towards a
slightly<br>
&quot;heavier&quot; transport that favors abstraction over speed.&nbsp; I think
that if we have<br>
situations where we need more speed, it will be situations like &quot;we need
to get<br>
4 GB of medical images from here to there&quot;, in which case we'll either
open<br>
up a &quot;raw&quot; AMQP channel that does no formatting, or, even faster,
open up<br>
a socket connection.<br>
<br>
I had written an initial version of a library to interface to AMQP that I am
mostly<br>
discarding in favor of a re-write that makes implementing servers easier and
provides<br>
more flexibility (for example: server classes that support messages where some
must be replied<br>
to and others that are not, direct messages, and fanout messages).<br>
<br>
I've got chunks of code written but am still working on the lower level
design.&nbsp; The<br>
architecture is pretty much done.&nbsp; Lots more coding to do.<br>
<br>
I've thought about this (very fun) problem a lot, and have had a few
&quot;epiphanies&quot;.&nbsp; There<br>
is a lot more details I'm not mentioning because there is so much.&nbsp; I
would be happy<br>
to answer any questions or trade ideas.<br>
<br>
PS: I don't always examine every rabbitmq-discuss message, so please feel free<br>
contact me directly.<br>
