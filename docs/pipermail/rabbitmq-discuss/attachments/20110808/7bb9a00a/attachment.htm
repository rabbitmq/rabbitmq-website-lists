Messages are processed in a fully asynchronous pipeline, and the decision on whether to discard a message is made at a later stage.<br>
<br>
Having this pipeline allows rabbit to absorb spikes in message load, i.e. it can read messages off the socket quickly and pass them to the next stage of the pipeline.<br>
<br>
However, if the load is not a spike and instead is consistently high then obviously rabbit will run out of memory eventually. To prevent that, memory usage is monitored and producers are temporarily blocked to relieve memory pressure. So...<div class="im">
