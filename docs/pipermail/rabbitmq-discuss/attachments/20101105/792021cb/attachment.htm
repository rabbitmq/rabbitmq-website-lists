<p class=MsoNormal>Overall we really like the new Management Plugin.&nbsp;
We're using this instead of Alice now.&nbsp; There are a few areas that could
use improvement though.&nbsp; Here are some feature requests:<o:p></o:p></p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal>- Queue Size (aggregate)<o:p></o:p></p>

<p class=MsoNormal>It would be helpful to display the current, aggregate queue
size, i.e. the total sum of all queues.&nbsp; This can help an Admin detect
when a queue is not getting drained.&nbsp; This could be displayed as a single
number on the /api/overview within the Message Rates row.<o:p></o:p></p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal>- RabbitMQ version<o:p></o:p></p>

<p class=MsoNormal>Can System Information display the RabbitMQ version?&nbsp; We
added this into Alice.&nbsp; It is very helpful when managing a cluster,
because sometimes a node fails to get upgraded.&nbsp; The version of Erlang can
also be useful.<o:p></o:p></p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal>- Uptime<o:p></o:p></p>

<p class=MsoNormal>Also added to Alice.&nbsp; This is useful to track
reliability.<o:p></o:p></p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal>-rabbitmqadmin CLI<o:p></o:p></p>

<p class=MsoNormal>We like it!&nbsp; However, it is missing the ability to display
/api/overview.&nbsp; Also would still prefer a key-value pair option.<o:p></o:p></p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal>-Firefox strangeness<o:p></o:p></p>

<p class=MsoNormal>I've noticed that I can only have one dashboard window
open.&nbsp; We have about 32 RabbitMQ nodes.&nbsp; Normally I would keep all 32
nodes open, each in a separate tab window, in Firefox.&nbsp; But this seems to slow
Firefox so much that it becomes unusable.&nbsp; At a certain point Firefox
needs to be killed and after that because of some glitch it is impossible to
login back into RabbitMQ's dashboard.<o:p></o:p></p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal>Great job overall though.&nbsp; The ability to drill down
into Channels and see individual rates and counts is really nice.<o:p></o:p></p>
