I find <a href="http://wiki.secondlife.com/wiki/Message_Queue_Evaluation_Notes">http://wiki.secondlife.com/wiki/Message_Queue_Evaluation_Notes</a> good.  They don&#39;t mention the tools you mention, however.  <div><br></div>

<div>You&#39;ve not given any detail to the type of app or use you have, and so it&#39;s difficult to compare them for you. QWIDS seems to be an odd choice as CORBA went out with the 90&#39;s for good reason.  Both Spread and QWIDS seem more like a message bus, as opposed to a message broker.</div>

<div><br></div><div>In spread you tell a program to join a spread group, then you send messages and everything in that group gets the message and processes it.  In RabbitMQ there are multiple workflows.  You can send messages with routing information, queue up messages for later consumption, use client side throttling to only receive messages when you need them, etc.</div>

<div><br></div><div>Honestly you should look at your problem area, what you&#39;re going to use RabbitMQ for, and if you need something like RabbitMQ or you need something like Spread.  I don&#39;t seem them as competing for the same space.<div>
