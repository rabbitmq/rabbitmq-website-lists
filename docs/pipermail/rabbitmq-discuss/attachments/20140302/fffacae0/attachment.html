</div>You can do #2 and #3 via AMQP but #3 will change message ordering:<br>
<br>
 * Response to Channel#QueueDeclare will contain (ready) message count<br>
 * Fetch queue head with Channel#BasicGet and requeue it. Note that this can change order of messages in the queue.<br>
<br>
In general, #2 and #3 should be done over AMQP, so using RabbitMQ .NET client.<br>
<br>
Number of queues can only be retrieved over HTTP API, e.g. using EasyNetQ’s client for it.<br>
