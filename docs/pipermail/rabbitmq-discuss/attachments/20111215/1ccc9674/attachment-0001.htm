<p class="MsoNormal">We have an issue with a .net client application that creates a connection in order to publish message and caches this connection in a variable. The problem is that when the program exits, it gets stuck in memory because there is an AMQP
 thread that doesn&#8217;t exit.<o:p></o:p></p>
<p class="MsoNormal">I&#8217;m not sure how the RabbitMQ threads are created but in .net there is an option to set a thread as a background thread and in this case when the application closes it will not wait for background threads to stop. Is the .net client setting
 the background threads it creates background? If not, can you change that?<o:p></o:p></p>
