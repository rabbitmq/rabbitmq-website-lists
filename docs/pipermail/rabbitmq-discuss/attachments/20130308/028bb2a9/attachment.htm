Hi, Brian:<div><br></div><div>What do you see if you try:</div><div><br></div><div>ps auxwww | grep -y beam</div><div><br></div><div>or:</div><div><br></div><div><div>ps auxwww | grep -y rabbit</div></div><div><br></div><div>
The &quot;beam&quot; is the name of the Erlang VM that Rabbit runs in, analogous to &quot;java&quot; being the name of the JVM a Java application would be running in.  When you spot a running rabbit broker you&#39;ll seem a beam process (the name may vary slightly depending on platform) whose very long command line contains a bunch of parameters that will implicate rabbit.</div>
<div><br></div><div>If you &quot;kill -9&quot; that process you should have things shut down and in a known state where you can get back to starting/stopping with /sbin/service, run sudo&#39;ed in the usual way...</div><div>
