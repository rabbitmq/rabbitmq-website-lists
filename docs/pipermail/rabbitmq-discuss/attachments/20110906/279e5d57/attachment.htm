<p class=MsoNormal>Hello,<o:p></o:p></p>

<p class=MsoNormal><span lang=EN-US>Here is the thing&nbsp;: <o:p></o:p></span></p>

<p class=MsoNormal><span lang=EN-US>- I&#8217;ve got a producer P and a consumer
C. <o:p></o:p></span></p>

<p class=MsoNormal><span lang=EN-US>- Pattern is request-response.<o:p></o:p></span></p>

<p class=MsoNormal><span lang=EN-US><o:p>&nbsp;</o:p></span></p>

<p class=MsoNormal><span lang=EN-US>P constantly submit requests to consumers
and wait for an answer for a given amount of time. Should C take longer to
answer, P will consider there was a time-out and it just sends message N+1,
discarding any answer related to N. <o:p></o:p></span></p>

<p class=MsoNormal><span lang=EN-US><o:p>&nbsp;</o:p></span></p>

<p class=MsoNormal><span lang=EN-US>Thing is, if nothing is done, C will still
process all messages it receives (and send response that will be discarded).
This makes C accumulate messages in case they take an unusually long time to process.<o:p></o:p></span></p>

<p class=MsoNormal><span lang=EN-US><o:p>&nbsp;</o:p></span></p>

<p class=MsoNormal><span lang=EN-US>I suppose this is a common problem. I was
wondering if anyone had a best practice for this. Here are the ideas I got :<o:p></o:p></span></p>

<p class=MsoNormal><span lang=EN-US>- P creates a different response queue for
each message, C checks whether the response queue exists before processing a
message<o:p></o:p></span></p>

<p class=MsoNormal><span lang=EN-US>- Assuming P and C are time synchronized, P
sends an expiration date along with each message, that C will check before
doing anything<o:p></o:p></span></p>

<p class=MsoNormal><span lang=EN-US><o:p>&nbsp;</o:p></span></p>

<p class=MsoNormal><span lang=EN-US>Any better idea would be appreciated,<o:p></o:p></span></p>

<p class=MsoNormal><span lang=EN-US><o:p>&nbsp;</o:p></span></p>

<p class=MsoNormal><span lang=EN-US>Thanks,<o:p></o:p></span></p>
