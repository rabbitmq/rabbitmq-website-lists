<div dir="ltr">This weekend, I upgraded one of our RabbitMQ clusters from 3.1.5 to 3.2.3.<div><br></div><div>After the upgrade, the management API has become somewhat unstable. Somewhere around 20% of the time, requests to the API stall. For example, a simple curl of the aliveness test endopint stalls after the GET request is sent (as seen via tcpdump). The management plugin simply never responds.</div>
