<p class="MsoNormal">Hi All,<o:p></o:p></p>
<p class="MsoNormal"><o:p>&nbsp;</o:p></p>
<p class="MsoNormal">We are facing problem in RabbitMQ clustering on two existing node enable for SSL. First we have set up SSL on both the nodes on SSL default port 5671 and then we have applied cluster set up with two disk nodes. After successful cluster
 set up we can able to see the cluster environment on management console which shows two disk nodes clustered.<o:p></o:p></p>
<p class="MsoNormal"><o:p>&nbsp;</o:p></p>
<p class="MsoListParagraph" style="text-indent:-18.0pt;mso-list:l1 level1 lfo1"><![if !supportLists]><span style="mso-list:Ignore">1.<span style="font:7.0pt &quot;Times New Roman&quot;">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
</span></span><![endif]>But after few hours While verifying it, we observed the management console doesn&#8217;t appear so we checked the services and its stops on both the machines.
<o:p></o:p></p>
<p class="MsoListParagraph"><o:p>&nbsp;</o:p></p>
<p class="MsoListParagraph">So we have started service of first node and then we can see the management console and it shows the second server disk node gets down. The time duration between set up and testing was hardly 1 hr. 30 mins. Node didn&#8217;t went up by
 following command. We followed attached installation document for RabbitMQ installation. We also checked the cookies files and its correct.<o:p></o:p></p>
<p class="MsoListParagraph"><i>rabbitmqctl start_app</i><o:p></o:p></p>
<p class="MsoListParagraph"><i>rabbitmq-server -detached</i>.<o:p></o:p></p>
<p class="MsoListParagraph" style="text-indent:-18.0pt;mso-list:l1 level1 lfo1"><![if !supportLists]><span style="mso-list:Ignore">2.<span style="font:7.0pt &quot;Times New Roman&quot;">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
</span></span><![endif]>To manage the cluster set up, we have created another disk node on the machine with diff name and same port (5671). The node is created successfully and shown the active status in management console. But the node working is not correct
 and is as follows.<o:p></o:p></p>
<p class="MsoListParagraph" style="margin-left:90.0pt;text-indent:-18.0pt;mso-list:l0 level1 lfo2">
<![if !supportLists]><span style="mso-list:Ignore">1.<span style="font:7.0pt &quot;Times New Roman&quot;">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
</span></span><![endif]>The messages are not transport properly on the queues generated after clustering. The queues are mirror queues. The queue status also shows &#8220;?&#8221; and when any message send to the queue, the management console is not able to open the queue
 detail window and the page gets hanged. It seems it goes into some loop. When we restart the RabbitMQ service, the page display properly but the queue status is same.
<o:p></o:p></p>
<p class="MsoListParagraph" style="margin-left:90.0pt;text-indent:-18.0pt;mso-list:l0 level1 lfo2">
<![if !supportLists]><span style="mso-list:Ignore">2.<span style="font:7.0pt &quot;Times New Roman&quot;">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
</span></span><![endif]>When we shut down the first node and shift the focus on 2<sup>nd</sup> node then the messages get deleted from all the queues and not able to retrieve the messages. The queues are persistence and durable queues. When we send the message
 on 2<sup>nd</sup> node then the behaviours of queue is same as mentioned in #1.<o:p></o:p></p>
<p class="MsoListParagraph" style="text-indent:-18.0pt;mso-list:l1 level1 lfo1"><![if !supportLists]><span style="mso-list:Ignore">3.<span style="font:7.0pt &quot;Times New Roman&quot;">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
</span></span><![endif]>If any node of RabbitMQ or its service get shut down abruptly then is there any provision to send notifications to administrator? Currently this functionality is exists for other Windows services
<o:p></o:p></p>
<p class="MsoNormal"><o:p>&nbsp;</o:p></p>
<p class="MsoNormal">If anybody knows how to start the shutdown node then kindly let us know.<o:p></o:p></p>
