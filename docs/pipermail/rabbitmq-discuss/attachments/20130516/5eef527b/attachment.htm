I think you are using &lt; 3.1.0 - in which case the throughput is the current instantaneous rate, and the absolute counts are for all currently open channels (so they can go backward when a channel is closed).<br>
<br>
In 3.1.0 this was rewritten. The absolute counts are now for all time (well, since the statistics database started) and you can specify sample_age and sample_incr parameters to get historical data - see<br>
