<br>
There are no immediate plans to support seamless upgrades across major<br>
versions.<br>
<br>
In general only upgrades between patch versions (e.g. 3.0.x to 3.0.y)<br>
can be performed without downtime. The exceptions are that 3.1.4 and<br>
3.1.5 are not compatible with other older 3.1.x versions in the same<br>
cluster, and 3.0.0 is not compatible with newer versions from the 3.0.x<br>
series.<br>
See <a href="https://www.rabbitmq.com/clustering.html#upgrading" target="_blank">https://www.rabbitmq.com/clustering.html#upgrading</a><br>
