It&#39;s probably not possible to make libtool 2.7 work.  We know that the<br>
libtool support for building with the Microsoft tool chain is broken in<br>
some libtool versions after 2.2.7.  Judging from the error you get, 2.7<br>
is one of the versions where it is broken.  If you don&#39;t like this, you<br>
may want to send your feedback to the libtool maintainers.<br>
<br>
This is why the README.windows file says:<br>
<br>
    [...] The GNU build tools have limited support for Microsoft<br>
    toolchain, but the install-mingw.sh script will install versions of<br>
    the packages that are known to be suitable.  In particular, only<br>
    libtool version 2.2.7a is known to work; later versions have been<br>
    reported to introduce problems.<br>
