<p class=MsoNormal><span lang=EN-US style='font-size:11.0pt;font-family:"Calibri","sans-serif";
color:#1F497D'>At web site there is such note:<o:p></o:p></span></p>

<p class=MsoNormal><span lang=EN-US style='font-size:11.0pt;font-family:"Calibri","sans-serif";
color:#1F497D'><o:p>&nbsp;</o:p></span></p>

<p class=MsoNormal><span lang=EN-US>Our goal is to streamline the broker
installation process such that you can have RabbitMQ up and running within two
minutes of completing your download. If this doesn't happen to you, please let
us know at <a href="mailto:legitimategrievance@rabbitmq.com">legitimategrievance@rabbitmq.com</a><o:p></o:p></span></p>

<p class=MsoNormal><span lang=EN-US><o:p>&nbsp;</o:p></span></p>

<p class=MsoNormal><span lang=EN-US style='font-size:11.0pt;font-family:"Calibri","sans-serif";
color:#1F497D'>I had some problems to install and launch it on windows 2008
server (64bit), but figured out, so I can share my experience:<o:p></o:p></span></p>

<p class=MsoNormal><span lang=EN-US style='font-size:11.0pt;font-family:"Calibri","sans-serif";
color:#1F497D'><o:p>&nbsp;</o:p></span></p>

<p class=MsoListParagraph style='text-indent:-18.0pt;mso-list:l0 level1 lfo1'><![if !supportLists]><span
lang=EN-US style='font-size:11.0pt;font-family:"Calibri","sans-serif";
color:#1F497D'><span style='mso-list:Ignore'>1.<span style='font:7.0pt "Times New Roman"'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
</span></span></span><![endif]><span lang=EN-US style='font-size:11.0pt;
font-family:"Calibri","sans-serif";color:#1F497D'>Because &nbsp;erlang is 32bit
app, it is installed by default to C:\Program Files (x86). I had problems changing
path in bat files (for unknown reason it didn&#8217;t found erl even if I
specified other location), so &nbsp;it was easier to reinstall it to C:\Program
Files folder which is default.<o:p></o:p></span></p>

<p class=MsoListParagraph style='text-indent:-18.0pt;mso-list:l0 level1 lfo1'><![if !supportLists]><span
lang=EN-US style='font-size:11.0pt;font-family:"Calibri","sans-serif";
color:#1F497D'><span style='mso-list:Ignore'>2.<span style='font:7.0pt "Times New Roman"'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
</span></span></span><![endif]><span lang=EN-US style='font-size:11.0pt;
font-family:"Calibri","sans-serif";color:#1F497D'>I installed rabbitmq as
windows service and started it, but it wasn&#8217;t working because service was
running as &#8220;Local System&#8221; account. I changed to log on as &#8220;Network
Service&#8221;, but the problem persisted.<o:p></o:p></span></p>

<p class=MsoListParagraph><span lang=EN-US style='font-size:11.0pt;font-family:
"Calibri","sans-serif";color:#1F497D'>&#8220;Network Service&#8221; by default
has not IO permissions to access windows directory (needed for erlang.cookie)
and your user account appdata directory for storing logs and db. So I changed
the bat script and specified another dir and added permissions to this folder.
Also you need to add permissions for c:\windows catalog because when rabbitmq
starts, it creates .erlang.cookie there (haven&#8217;t found a successful way
to change this path). After that, you could remove permissions from folder and
add it only to that file.<o:p></o:p></span></p>

<p class=MsoListParagraph style='text-indent:-18.0pt;mso-list:l0 level1 lfo1'><![if !supportLists]><span
lang=EN-US style='font-size:11.0pt;font-family:"Calibri","sans-serif";
color:#1F497D'><span style='mso-list:Ignore'>3.<span style='font:7.0pt "Times New Roman"'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
</span></span></span><![endif]><span lang=EN-US style='font-size:11.0pt;
font-family:"Calibri","sans-serif";color:#1F497D'>After I changed paths in bat
file, it was using the old ones, so finally I removed all &#8220;if &#8230;&#8221;
and epically specified new paths.<o:p></o:p></span></p>

<p class=MsoNormal><span lang=EN-US style='font-size:11.0pt;font-family:"Calibri","sans-serif";
color:#1F497D'><o:p>&nbsp;</o:p></span></p>

<p class=MsoNormal><span lang=EN-US style='font-size:11.0pt;font-family:"Calibri","sans-serif";

color:#1F497D'>Changed to:<o:p></o:p></span></p>

<p class=MsoNormal><span lang=EN-US style='font-size:11.0pt;font-family:"Calibri","sans-serif";
