exchange-to-exchange bindings are great for complex topic routing -- but this assumes the routing key for each message is (a) decided on entry and (b) stays the same throughout a message&#39;s flow through the system.<br><br>
so, what happens if a consumer needs to affect a particular message&#39;s route? e.g. to reinject/rebroadcast invalid/malformed/unauthorized messages into a different queue/pipeline. Sure, it could change the routing key, but it&#39;d still need to rebroadcast/reinject, right?<br>
