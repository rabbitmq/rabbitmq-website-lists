<br>
It depends on what kind of web hosting account your have. You will not<br>
be able to run RabbitMQ on a web hosting service that does not allow<br>
running Erlang applications.<br>
<br>
If you have remote console access and you are able to open access to TCP<br>
port 5672 for AMQP clients then you can follow the installation<br>
instructions as if the server was local:<br>
<a href="http://www.rabbitmq.com/download.html" target="_blank">http://www.rabbitmq.com/download.html</a><br>
<br>
If your infrastructure is provided by Amazon EC2 then these notes might<br>
be helpful:<br>
<a href="http://www.rabbitmq.com/ec2.html" target="_blank">http://www.rabbitmq.com/ec2.html</a><br>
