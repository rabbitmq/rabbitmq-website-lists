    Hmn. If you set these net_kernel parameters for an Erlang node, then
    you need to restart epmd on that host IIRC. The routing is usually
    thus<br>
    <br>
    $ rabbitmqctl stop<br>
    $ epmd -names<br>
    epmd: up and running on port 4369 with data:<br>
    $ epmd -kill<br>
    <br>
    Then restart your rabbit (on that node!) and you *should* be good to
    go. Note that if `epmd -names` returns some running erlang nodes on
    that host then you'll need to stop/kill them before stopping epmd
    itself.<br>
