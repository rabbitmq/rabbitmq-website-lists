<p>You would only need to know the number of subtasks at each stage. You would not need to know the total number of subtasks that would be created during the whole run.</p>
<p>Example... start job, make 10 subtasks, give a weight of 0.1. The total weight is 1. Now suppose one subtask comes back and needs to be split into 4 subtasks. Give each of those weight 0.025. The grand total is still 1; indeed it is invariant<br>
