<div style><br></div><div style><div>  <a href="http://192.168.34.56:43556">192.168.34.56:43556</a> &lt;--&gt; <a href="http://10.1.2.3:5672">10.1.2.3:5672</a></div><div><div>  <a href="http://172.16.99.104:50602">172.16.99.104:50602</a> &lt;--&gt; <a href="http://10.1.2.3:5672">10.1.2.3:5672</a></div>
</div><div><div>  <a href="http://10.202.14.66:61082">10.202.14.66:61082</a> &lt;--&gt; <a href="http://10.1.2.3:5672">10.1.2.3:5672</a></div></div><div><br></div><div style>and so on. In each case, the IP/port of the server&#39;s end of the socket is the same. This means that you&#39;re not limited to 65k TCP connections per interface; instead, you&#39;re limited to approximately 2^48 = 281,474,976,710,656 connections *per listening socket* :-)</div>
