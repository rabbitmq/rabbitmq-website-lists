<div dir="ltr">True - I stand corrected.<br><br>For the purposes of this argument it doesn&#39;t really matter, because the persister log is still in the mnesia directory - the point is that the directory should be moved to a separate disk to partition the I/O. If mnesia tables have a great deal of I/O as well, then the persister log itself should be moved somewhere else. The point is also that I/O (and memory, and CPU) should be measured if performance is a concern.<br>
