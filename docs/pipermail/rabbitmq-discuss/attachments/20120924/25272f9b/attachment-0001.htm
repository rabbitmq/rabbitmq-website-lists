    <br>
    No, there is no kind of 'attachments' API in AMQP like the one in
    soap. If you want to 'know' something about the content in the
    consumer then you can set headers in the message yourself and handle
    these explicitly, dealing with stuff like content-type, encoding,
    mime-type, etc. It's all under your control, as the body and headers
    are treated as opaque by the broker.<br>
