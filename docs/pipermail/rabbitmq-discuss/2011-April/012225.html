On 01/04/11 00:54, Jason J. W. Williams wrote:
> Are there any known gotchas for running Shovel in a RabbitMQ cluster?
> For example is it advised to only run shovel on one node of the
> cluster?

Each node that runs shovel is independent, so if you were to set up 
several nodes with the same config you'd end up with multiple shovels 
doing the same thing, which may not be what you want (message ordering). 
So as in so many other cases, It Depends What You Want To Do.
