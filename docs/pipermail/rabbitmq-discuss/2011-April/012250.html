...but since you start rabbit with 'sudo', it's the 'root' user that 
needs to have 'erl' on the path.

Btw, you really shouldn't run rabbit as root. As explained at 
http://www.rabbitmq.com/install.html#running-generic-unix, just set the 
RABBITMQ_MNESIA_BASE and RABBITMQ_LOG_BASE vars to point to locations 
which are writeable by the current user and then run 
sbin/rabbitmq-server without sudoing.
