On 21/06/12 13:48, José Micó wrote:
> On 06/21/2012 08:45 AM, Simon MacMullen wrote:
>> - Debian: uninstalling failed if broker was stopped
> 
> This may still be failing. First attempt to upgrade to 2.8.3 on Debian 
> fails if the server is stopped:

Version 2.8.2 of the Debian package would fail to uninstall if the
broker was not running. That is the problem you are running into. The
workaround is to ensure that the broker is running when you uninstall or
upgrade.

You will not run into the same problem when upgrading after version
2.8.3, which is what the fix provides. The released fix does not apply
to version 2.8.2 retro-actively.
