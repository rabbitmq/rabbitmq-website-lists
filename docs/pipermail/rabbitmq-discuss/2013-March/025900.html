That fix is in 2.7.1 (and indeed all versions >= 2.6.0) but you're not 
seeing that problem - you are getting eaddrinuse, i.e. the port is 
already taken.

Later versions of RabbitMQ than 2.7.1 report this error in a more 
readable way :-)

Cheers, Simon
